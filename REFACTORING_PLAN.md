## UI refactoring plan

Цель: **минимизировать кастомные стили и дублирование кода**, максимально полагаясь на Element Plus и несколько переиспользуемых layout/UI‑компонентов.

### ✅ 1. Глобальные стили и токены

- Проанализировать `src/styles/**` на:
  - неиспользуемые классы,
  - дублирующиеся паттерны (flex‑лейауты, контейнеры `__content`, `__loading` и т.п.),
  - переопределения Element Plus, которые можно убрать или унифицировать.
- Ввести 2–4 общих layout‑класса (например, `.layout-main`, `.layout-section`, `.layout-loading-center`) и заменить локальные дубли.

### ✅ 2. Лоадеры и async‑состояния

- Стандартизировать отображение состояний `loading / error / empty / list` во всех view:
  - использовать общий компонент (например, `AsyncSection`) или строгий паттерн с `ContentLoader`/`v-loading` + `el-empty`.
- Оставить `ContentLoader.vue` единственной точкой для inline‑лоадеров (иконка + текст).
- Убедиться, что нет других самодельных спиннеров/сообщений загрузки.

### ✅ 3. Infinite scroll и пагинация

- Унифицировать использование `InfiniteScrollLoadMore.vue`:
  - единый API `hasNextPage`, `isFetchingNextPage`, `@loadMore`.
- Рассмотреть переход на директиву `v-infinite-scroll`, если UX позволяет.
- Убрать локальные реализации «Load more», заменить на общий компонент/паттерн.

### ✅ 4. Layout‑паттерны страниц

- Каталоги (movies/series/books/comedy/music):
  - выделить общий layout (заголовок + `el-main` + loading/empty/list),
  - либо через компонент `CatalogLayout`, либо через общие классы в `catalog.css`.
- Контент‑страницы (Movie/Series/Book/Music/Comedy Content):
  - найти повторяющиеся шаблоны (заголовок, мета, основной блок),
  - вынести в общий контейнер (используем `ContentPageFrame`).

### ✅ 5. Хедер и навигация

- `AppHeader.vue`:
  - рассмотреть перевод nav‑линков на `el-menu` с `router`‑режимом **или** общий компонент `NavLink`,
  - убрать дублирующуюся логику определения активного пути.

### ✅ 6. Повторяющиеся UI‑паттерны

- Карточки:
  - убедиться, что все базовые карточки используют `el-card` и минимальный CSS,
  - вынести общие паттерны (header/footer/иконки) в утилитарные классы или маленькие компоненты.
- Кнопки‑иконки/тулбары:
  - найти повторяющиеся маленькие икон‑кнопки и привести их к общему виду (через `el-button` + 1–2 общих класса или `IconActionButton`).

### ✅ 7. Phrase drawer и словарь

- Почистить legacy‑стили вокруг `PhraseDrawer`:
  - удалить неиспользуемые классы из `phrase-drawer.css`,
  - убедиться, что `PhraseDrawer*` используют общие паттерны карточек и лоадеров.
- В словаре:
  - выровнять `DictionaryEntryCard` по общему стилю карточек,
  - если повторяется разметка фразы/перевода/комментария — вынести в отдельный маленький компонент.

### ✅ 8. Финальная чистка CSS

- Поиск неиспользуемых классов в `src/styles/**/*.css` и scoped‑стилях компонентов.
- Сведение повторяющихся CSS‑правил к общим утилити‑классам.
- Финальный прогон `npm run build` и визуальная проверка основных экранов.
